# منصة متابعة منظومة اجتماعات نائب الأمين

> نظام ERP داخلي لإدارة منظومة اجتماعات نائب الأمين للبنية التحتية — أمانة منطقة المدينة المنورة.

يحوّل ملف Excel الحالي إلى قاعدة بيانات فعلية قابلة للتحديث الدوري، ويدعم متابعة الاجتماعات،
المهام، المسؤوليات، الإفادات، التصعيد، الاعتماد، الإغلاق، المؤشرات، والتقارير التنفيذية، مع نظام
صلاحيات يعتمد على **جهة الاجتماع** و**المسؤول المكلف** معًا، ودعم كامل للمهام المشتركة وسير العمل الرسمي.

## ▶️ التشغيل السريع (H2)

المتطلبات: **JDK 21** و **Maven** مثبتان.

```bash
cd meetings-erp
mvn spring-boot:run
```

أو عبر ملف JAR:

```bash
mvn -DskipTests package
java -jar target/meetings-erp.jar
```

ثم افتح المتصفح على: **http://localhost:8080**

> تُنشأ قاعدة بيانات H2 تلقائيًا في مجلد `./data` مع بيانات أولية (جهات، مستخدمون، عينة اجتماعات ومهام).
> لوحة H2 متاحة على `http://localhost:8080/h2-console` (JDBC URL: `jdbc:h2:file:./data/meetingsdb`، المستخدم `sa` بدون كلمة مرور).

## 🗄️ التبديل إلى قاعدة إنتاجية

فعّل ملف الإعداد المناسب (`application-postgres.properties` أو `application-mysql.properties`)
وعدّل بيانات الاتصال ثم شغّل:

```bash
# PostgreSQL
mvn spring-boot:run -Dspring-boot.run.profiles=postgres

# MySQL
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
```

## 📥 استيراد ملف Excel (تحديث دوري)

1. سجّل الدخول كمدير نظام ← **استيراد Excel**.
2. ارفع ملف `.xlsx` (قاعدة بيانات منظومة الاجتماعات).
3. يقرأ النظام الأعمدة العربية تلقائيًا، وينظّف البيانات، ويوحّد الحالات، ويكشف:
   - الصفوف **الجديدة**، **المكررة**، و**التي بها أخطاء**.
4. تظهر **شاشة معاينة** بالإحصاءات والتفاصيل قبل الاعتماد.
5. عند الاعتماد: تُضاف البيانات إلى قاعدة البيانات (مع خيار **تحديث** المكرر بدل تخطيه).
6. يمكن **تنزيل ملف الأخطاء** (CSV).

**قاعدة منع التكرار:** محضر الاجتماع + تاريخ الاجتماع + المهمة المسندة + المسؤول + جهة الاجتماع.

**توحيد الحالات:** `لما يبدا/لم يبدا/لم يبدأ بعد → لم يبدأ`، `جاري/قيد الإنجاز → جاري العمل`،
`تم/منجز/مكتمل → تم الانتهاء`، `جزئي → تسليم جزئي`.

**نِسب الإنجاز:** تم الانتهاء = 100٪، تسليم جزئي = 50٪، جاري العمل = 30٪، لم يبدأ/متأخر = 0٪.

## 👥 الأدوار والصلاحيات

الصلاحيات تُطبَّق على معيارين معًا: **جهة الاجتماع** و**المسؤول المكلف**، مع دعم المهام المشتركة.

- **مدير النظام (ADMIN):** صلاحيات كاملة (مستخدمون، جهات، اجتماعات، مهام، استيراد واعتماد، حذف، تصعيد، إغلاق، تدقيق).
- **حساب جهة (DEPARTMENT):** يرى اجتماعات ومهام جهته + المهام المشتركة التي جهته طرف فيها، ويحدّث إفاداتها.
- **حساب مسؤول (RESPONSIBLE):** يرى المهام المسندة له + المشتركة التي يشارك فيها، ويحدّث حالتها وإفادتها.
- **مشاهد (VIEWER):** عرض وفلاتر وتقارير فقط دون تعديل.
- **متابعة تنفيذية (EXECUTIVE):** عرض جميع المهام + ملاحظات تنفيذية + اقتراح تصعيد.

## 🔄 سير العمل الرسمي للمهام

`إنشاء → إسناد → قيد التنفيذ → مراجعة الإفادة → اعتماد الإفادة → تسليم جزئي → تصعيد → متأخر → مكتمل → مغلق`

- لا يمكن إغلاق المهمة قبل اعتماد الإفادة أو اكتمالها.
- المهمة المشتركة لا تُغلق إلا باكتمال واعتماد إجراءات **جميع** الجهات المشاركة، أو باعتماد مدير النظام.
- المهام تُوسم **متأخرة تلقائيًا** عند تجاوز تاريخ الاستحقاق.
- كل تغيير حالة يُحفظ في **سجل سير العمل** باسم المستخدم والتاريخ، وكل إجراء جوهري في **سجل التدقيق**.
